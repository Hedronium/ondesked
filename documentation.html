Introduction

Ondesked is a markup language which is used for developing GUI for desktop apps.

It's based on XML, and is similar to HTML; if you know HTML, you already know some of Ondesked.

Every Ondesked document should have an app element as the root element. Just like this:

<app>...</app>

You can think of it as equivalent to an html tag.

Ondesked is designed to be strict, to prevent any unexpected behavior, and to keep codes consistent.

So, every Ondesked document should start with app tag, otherwise it just won't work.

App

App, as the name explains, represents the app that you're building.

As mentioned in the previous heading, theapp element should be at the root of the document.

Syntax:

<app>...</app>

Attributes:

  • idIdentifier of the app, extremely important even if it may not seem to be doing much.

Frame

A frame is pretty much a general window, much like the browser window you're reading this on right now.

Frames must go inside app tags. Here's the basic syntax:

<frame>...</frame>

Attributes:

  • titleTitle of the frame.

Classes:

  • centeredCenter the frame in the screen.
  • visibleMake the frame visible.

Panel

A panel holds UI elements, such as buttons, textfields, textareas, checkboxes, etc together. It may also hold other panels.

It's highly recommended to always use panels to hold UI elements, whenever possible.

Syntax:

<panel>...</panel>

Button

A button is an UI element which may be clickable.

Syntax:

<button>...</button>

Here, ... represents text you want to appear inside the button.

Attributes:

  • onclickName of the event which you want to be invoked when the user clicks on the button.

Textfield

An UI element where the user can type single-lined text.

Syntax:

<textfield>...</textfield>

Here, ... represents text you want to appear inside the textfield.

Textarea

Similar to Textfield but allows multi-lined input.

Syntax:

<textarea>...</textarea>

Here, ... represents text you want to appear inside the textarea.

Attributes:

  • widthHow wide the textarea will be in pixel.
  • heightHow tall the textarea will be in pixel.

Collapsible Pane

A UI element which holds UI elements, such as buttons, textfields, textareas, panels, etc, and can expand and collapse on click.

Syntax:

<colpane>...</colpane>

Here, ... represents its content - other elements it holds. The content will expand and collapse on click.

Table

An UI element which organizes data into rows and columns.

Syntax:

<table>
	<thead>
		<tr>
			<th>...</th>
			<th>...</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>...</td>
			<td>...</td>
		</tr>
	</tbody>
</table>

A table has a head (thead) and a body (tbody).

The head includes the column cells (th) , whilst the body includes the data cells (td).

Each tr element represents a row.

The ... includes textual content.

So here, we have a table with:

  • One row of two column cells.
  • One row of two data cells.
